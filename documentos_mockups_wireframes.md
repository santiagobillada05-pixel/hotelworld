**PLANIFICACIÓN DE INTERFACES DE USUARIO (UI)** 

## **Guía de Estructura Visual y Wireframes** 

**Proyecto:** Portal Web Hotel · **Formato:** mockups_wireframes.md 

Este documento detalla la distribución de componentes y bloques de información que configuran la interfaz de usuario del proyecto. Define de manera jerárquica los elementos que se implementarán a nivel de maquetación en los archivos HTML HTML correspondientes. 

## **1. Maquetación y Organización de la Página de Inicio (Home Page)** 

La pantalla principal se estructurará a través de secciones secuenciales claramente identificables: 

- **Bloque Superior de Navegación (Header):** Área fija que contendrá la identidad gráfica (Logo institucional  en  la  izquierda),  un  menú  de  navegación  centrado  o  alineado  a  la  derecha  con hipervínculos  directos  a  las  secciones  internas  (Inicio,  Habitaciones,  Servicios,  Contacto)  y  un elemento de llamada a la acción destacado para iniciar reservas de forma ágil. 

- **Sección de Bienvenida e Impacto (Hero Section):** Espacio de pantalla completa o de gran formato que integrará una imagen ambiental de alta definición, el eslogan principal del hotel y una barra de consulta rápida equipada con selectores de fechas de estancia y número de personas. 

- **Módulo de Bloques de Contenido Destacado:** Distribución en cuadrícula (Grid) para exhibir las tipologías de habitaciones más relevantes, empleando tarjetas de producto individuales con imagen, texto descriptivo, tarifa comercial base y acceso directo a la ampliación de detalles. 

- **Cierre de Página (Footer):** Zona técnica inferior dedicada a reflejar datos institucionales de contacto telefónico, dirección física verificada, mapa interactivo de ubicación, enlaces de redes sociales corporativas e hipervínculos hacia las normativas de reserva y cancelación. 

## **2. Diseño Funcional del Formulario de Entrada de Datos (Reservas)** 

El formulario se dispondrá dentro de un contenedor visual independiente con aislamiento de fondo para concentrar la atención del usuario en el proceso de captura de información: 

- **Fase 1: Configuración de la Estancia:** Controles web limpios para la introducción de las fechas calendarizadas  obligatorias  (Check-in  /  Check-out)  complementados  con  una  lista  desplegable parametrizada para la selección estricta del tipo de alojamiento. 

- **Fase 2: Registro de Identificación del Huésped:** Campos de texto estructurados con etiquetas claras  sobrepuestas  para  recopilar  Nombre  y  Apellido  completos,  dirección  válida  de  correo electrónico para notificaciones y teléfono móvil de contacto. 

- **Fase  3:  Notas  Adicionales:** Caja  de  texto  multilínea  opcional  para  anotaciones  especiales (requerimientos de accesibilidad, especificaciones de alimentación o solicitudes horarias).
